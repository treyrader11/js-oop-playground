üìö Challenge 1: Basic Class Creation

üéØ What You'll Learn

This challenge introduces the fundamental building blocks of JavaScript classes: constructors, properties, and methods.

üìñ Class Basics

A class is like a blueprint for creating objects. It defines:

  • Properties: Data that each object will have
  • Methods: Functions that each object can perform

üí° Example Structure

class Book {
  constructor(title, author, pages) {
    this.title = title;
    this.author = author;
    this.pages = pages;
  }

  getInfo() {
    return `${this.title} by ${this.author} - ${this.pages} pages`;
  }

  read() {
    return `You are reading ${this.title}`;
  }
}

üîë Key Concepts

  • constructor runs automatically when creating a new instance
  • this refers to the current instance of the class
  • Methods are functions defined inside the class
  • Use template literals (`${}`) to construct strings

üí° Pro Tip

Always use this.propertyName to access properties inside class methods. Without this, JavaScript won't know you're referring to the class property!

üîí Challenge 2: Class with Private Fields

üéØ What You'll Learn

Encapsulation is a core principle of object-oriented programming. Private fields allow you to hide internal data and control how it's accessed.

üìñ Private Fields

Private fields in JavaScript:

  • Start with a # symbol
  • Must be declared at the top of the class
  • Cannot be accessed from outside the class
  • Provide data protection and encapsulation

üí° Example Structure

class BankAccount {
  #balance; // Private field declaration

  constructor(accountHolder, initialBalance) {
    this.accountHolder = accountHolder;
    this.#balance = initialBalance;
  }

  deposit(amount) {
    this.#balance += amount;
    return this.#balance;
  }

  withdraw(amount) {
    if (amount > this.#balance) {
      return "Insufficient funds";
    }
    this.#balance -= amount;
    return this.#balance;
  }

  getBalance() {
    return this.#balance;
  }
}

üîë Key Concepts

  • Private fields are declared at the class level (before constructor)
  • Use # prefix every time you access the private field
  • Methods act as gatekeepers, controlling access to private data
  • This prevents direct manipulation of sensitive data

üí° Pro Tip

Private fields help prevent bugs by ensuring data can only be modified through your controlled methods. This is especially important for sensitive data like account balances!

üêï Challenge 3: Basic Inheritance

üéØ What You'll Learn

Inheritance allows you to create specialized classes based on more general ones. Child classes inherit properties and methods from their parents.

üìñ How Inheritance Works

  • Parent class contains common properties and methods
  • Child class extends the parent with extends
  • Child inherits all parent properties and methods automatically
  • Child can override parent methods to customize behavior
  • Child can add new methods specific to itself

üí° Example Structure

// Parent class
class Animal {
  constructor(name, species) {
    this.name = name;
    this.species = species;
  }

  makeSound() {
    return "Some generic animal sound";
  }

  introduce() {
    return `I am ${this.name}, a ${this.species}`;
  }
}

// Child class
class Dog extends Animal {
  constructor(name, breed) {
    super(name, "Dog"); // Call parent constructor
    this.breed = breed;
  }

  makeSound() {
    return "Woof! Woof!"; // Override parent method
  }

  getBreed() {
    return this.breed; // New method
  }
}

üîë Key Concepts

  • extends keyword creates the inheritance relationship
  • super() must be called before using this in child constructor
  • Overriding means providing a new implementation
  • Child class has access to all parent methods unless overridden

üí° Pro Tip

Think of inheritance as a "is-a" relationship. A Dog is an Animal, so it makes sense for Dog to extend Animal and inherit its general behaviors!

‚ö° Challenge 4: Inheritance with Super

üéØ What You'll Learn

The super keyword is your gateway to the parent class. It lets you call parent constructors and methods while adding new functionality.

üìñ Understanding Super

The super keyword has two main uses:

  • In constructor: super(args) calls the parent constructor
  • In methods: super.methodName() calls parent methods

üí° Example Structure

class Vehicle {
  constructor(make, model, year) {
    this.make = make;
    this.model = model;
    this.year = year;
  }

  getAge() {
    return 2025 - this.year;
  }

  getInfo() {
    return `${this.year} ${this.make} ${this.model}`;
  }
}

class ElectricCar extends Vehicle {
  constructor(make, model, year, batteryCapacity) {
    super(make, model, year); // Call parent constructor
    this.batteryCapacity = batteryCapacity;
  }

  getInfo() {
    // Call parent method and extend it
    return `${super.getInfo()} (Electric, ${this.batteryCapacity}kWh battery)`;
  }

  getBatteryInfo() {
    return `Battery capacity: ${this.batteryCapacity}kWh`;
  }
}

üîë Key Concepts

  • super() must be called before accessing this
  • Use super.method() to reuse parent functionality
  • You can combine parent behavior with new functionality
  • This promotes code reuse and maintains the inheritance chain

üí° Pro Tip

When overriding methods, use super.methodName() to keep the parent's logic and just add to it. This is better than rewriting everything from scratch!

üî∑ Challenge 5: Multi-Level Inheritance & Polymorphism

üéØ What You'll Learn

Multi-level inheritance creates a hierarchy of classes, each adding more specific functionality. Polymorphism allows different classes to implement the same methods in their own way.

üìñ Inheritance Hierarchy

In this challenge, you'll create three levels:

  • Shape (Grandparent): Most general, basic structure
  • Rectangle (Parent): Extends Shape, adds specific calculations
  • Square (Child): Extends Rectangle, specialized rectangle

üí° Example Structure

class Shape {
  constructor(name) {
    this.name = name;
  }

  getArea() {
    return 0; // Default implementation
  }

  getPerimeter() {
    return 0;
  }

  describe() {
    return `This is a ${this.name} with area ${this.getArea()} and perimeter ${this.getPerimeter()}`;
  }
}

class Rectangle extends Shape {
  constructor(width, height) {
    super("Rectangle");
    this.width = width;
    this.height = height;
  }

  getArea() {
    return this.width * this.height;
  }

  getPerimeter() {
    return 2 * (this.width + this.height);
  }
}

class Square extends Rectangle {
  constructor(side) {
    super(side, side); // A square is a rectangle with equal sides
  }

  describe() {
    return `This is a Square with side ${this.width}, area ${this.getArea()} and perimeter ${this.getPerimeter()}`;
  }
}

üîë Key Concepts

  • Polymorphism: Each class provides its own implementation of methods
  • Inheritance chain: Square inherits from Rectangle, which inherits from Shape
  • Methods can be overridden at any level
  • describe() automatically uses the correct getArea() and getPerimeter() for each class

üí° Pro Tip

Notice how Square doesn't need to override getArea() or getPerimeter() because it inherits Rectangle's implementations, which work perfectly for a square! This is smart code reuse.

‚≠ê Bonus: Static Methods & Getters/Setters

üéØ What You'll Learn

Advanced class features that make your code more elegant and powerful: static methods for class-level functionality and getters/setters for computed properties with validation.

üìñ Static Methods

  • Belong to the class itself, not instances
  • Called on the class: Temperature.freezingPoint()
  • Cannot access instance properties (no this)
  • Perfect for factory methods and utility functions

üìñ Getters and Setters

  • Look like properties but act like methods
  • Getters: access with obj.property (no parentheses)
  • Setters: assign with obj.property = value
  • Allow validation and computed values

üí° Example Structure

class Temperature {
  #celsius;

  constructor(celsius) {
    this.#celsius = celsius;
  }

  // Getter - access like: temp.celsius
  get celsius() {
    return this.#celsius;
  }

  // Setter - use like: temp.celsius = 25
  set celsius(value) {
    if (value < -273.15) {
      throw new Error("Temperature below absolute zero!");
    }
    this.#celsius = value;
  }

  // Computed property with getter
  get fahrenheit() {
    return (this.#celsius * 9/5) + 32;
  }

  // Setter converts Fahrenheit to Celsius
  set fahrenheit(value) {
    this.celsius = (value - 32) * 5/9;
  }

  // Static method - call with Temperature.freezingPoint()
  static freezingPoint() {
    return new Temperature(0);
  }

  static boilingPoint() {
    return new Temperature(100);
  }
}

üîë Key Concepts

  • Use get keyword before method name for getters
  • Use set keyword before method name for setters
  • Use static keyword for class-level methods
  • Getters/setters are accessed like properties, not methods
  • Static methods create instances or provide utilities

üí° Pro Tip

Getters and setters make your API cleaner. Instead of temp.getCelsius() and temp.setCelsius(25), you can use temp.celsius and temp.celsius = 25. Much more intuitive!
